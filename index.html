Merge Sort

March 22, 2014 by Sreeprasad

This blog post shows explains Merge Sort

Merge sort is a classic example of divide and conquer approach to solve sorting problem. My knowledge about Sorting came mostly from professor Erik Demaine lecture on Intro to Algorithm. I found it interesting and easy to understand. I also found Princeton lectures on MergeSort extremely easy to understand.

Merge sort does can explained in 3 steps

  1. divide the input part(s)
  2. conquer(solve) the parts recursively
  3. Combine(merge) the result(s) to solve the original


Algorithm
We divide the array into 2 parts and sort each part recursively. We stop only when the array size is 1 When the array size is 1 then no work is done because every sequence is in sorted order.Then we start merging the two arrays each of which has size 1. This part is easy as we need to compare only 2 elements. Merge is done by calling auxiliary procedure called Merge.

merge(A, p, q, r)

merge(A, p, q, r) where A is an array and p, q, and r are indices into the array such that p≤q < r. This ofcourse assumes that subarray A[p...q] and A[q+1...r] are sorted.
This merging takes Θ(n) time where n=r-p+1
